## 1. 摩尔投票算法（Boyer-Moore Majority Vote）

### 1.1 核心思想
- 想象一个擂台比武：候选擂主有一个“生命值”。
- 遍历数组：
  - 如果生命值为 0，则当前元素成为新擂主，生命值置 1。
  - 否则，如果当前元素等于擂主，则生命值 +1（同门加血）；否则生命值 -1（不同则抵消）。
- 最终留下的擂主，如果数组中存在**出现次数超过一半**的元素，那么它就是该元素。

### 1.2 正确性原理（配对抵消）
- 将不同元素视为不同阵营的士兵。
- 每次让两个不同阵营的士兵同归于尽（抵消一次）。
- 由于多数阵营的士兵数量超过其他所有阵营的总和，无论怎么抵消，最后剩下的士兵一定属于多数阵营。

### 1.3 经典代码（C++）
```cpp
int majorityElement(vector<int>& nums) {
    int candidate = 0, count = 0;
    for (int x : nums) {
        if (count == 0) {
            candidate = x;
            count = 1;
        } else if (x == candidate) {
            count++;
        } else {
            count--;
        }
    }
    // 题目保证多数元素存在，直接返回 candidate
    return candidate;
}
```
- 时间复杂度：O(n)
- 空间复杂度：O(1)

## 2. 题目特殊性（LeetCode 961）

### 2.1 题目条件
- 数组长度 = 2n
- 有 n+1 个不同元素，其中 n 个元素只出现一次，1 个元素出现 n 次
- 即目标元素出现次数 = n，恰好是总长度的一半

### 2.2 为什么不能直接用摩尔投票
- 摩尔投票要求目标元素出现次数 **严格超过一半**（> n/2）。
- 这里目标元素出现次数等于一半（= n/2），不满足条件。如果直接使用，可能返回其他元素。

## 3. 方法二：摩尔投票 + 分类讨论

### 3.1 转化思路
通过分类讨论，将问题转化为经典摩尔投票可以解决的形式：

- **情况 A**：`nums[0]` 就是目标元素  
  那么它一定会在 `[1, n-1]` 中再次出现（因为一共出现 n 次，除了下标 0 还有 n-1 次）。  
  因此，只要在遍历时发现某个 `nums[i] == nums[0]`，就可以直接返回。

- **情况 B**：`nums[0]` 不是目标元素  
  那么 `nums[0]` 是那 n 个唯一元素之一。去掉它之后，剩余数组长度为 `2n-1`，其中目标元素仍出现 n 次，其他元素共 `n-1` 次。  
  此时目标元素出现次数 **n > (2n-1)/2**，即成为绝对众数，可以用经典摩尔投票找出。

### 3.2 代码实现（C++）
```cpp
int repeatedNTimes(vector<int>& nums) {
    int ans = 0, hp = 0;
    for (int i = 1; i < nums.size(); i++) {
        int x = nums[i];
        // 情况 A：nums[0] 是目标元素
        if (x == nums[0]) {
            return x;
        }
        // 情况 B：对剩余部分进行摩尔投票
        if (hp == 0) {
            ans = x;
            hp = 1;
        } else {
            hp += (x == ans) ? 1 : -1;
        }
    }
    // 如果情况 A 未触发，则 ans 即为目标元素
    return ans;
}
```

### 3.3 为什么这样可行
- 循环从 `i = 1` 开始，将 `nums[0]` 作为“哨兵”。
- **同时处理两个分支**：
  1. 如果遇到等于 `nums[0]` 的元素，说明情况 A 成立，立即返回。
  2. 否则，执行经典摩尔投票，但投票的对象是从 `nums[1]` 到末尾的序列（即去掉 `nums[0]` 后的序列）。
- 如果情况 A 没有触发，说明 `nums[0]` 不是目标元素，那么摩尔投票结束后 `ans` 就是剩余部分中的绝对众数，即答案。

### 3.4 复杂度分析
- 时间复杂度：O(n)，只需一次遍历。
- 空间复杂度：O(1)，仅用两个变量。

## 4. 方法亮点与总结

| 方面           | 说明                                                          |
| -------------- | ------------------------------------------------------------- |
| **算法核心**   | 摩尔投票（抵消思想）                                          |
| **问题难点**   | 出现次数恰好一半，不是严格多数                                |
| **解决方案**   | 利用 `nums[0]` 作哨兵，分类讨论转化为经典问题                 |
| **代码技巧**   | 一次循环同时完成“检查 `nums[0]` 是否重复”和“剩余部分摩尔投票” |
| **时间复杂度** | O(n)                                                          |
| **空间复杂度** | O(1)                                                          |

### 4.1 适用场景
- 需要 O(1) 空间且数组长度较大时。
- 问题本身是“寻找出现次数为一半的元素”，且其他元素各出现一次（保证转化后成为绝对众数）。

### 4.2 扩展思考
- 如果题目改成“出现次数超过 n/3”或“恰好 n/2 但无其他限制”，摩尔投票也有相应变种。
- 这种“用第一个元素作哨兵”的思路，在需要处理边界条件时值得借鉴。

---

## 5. 总结

**摩尔投票 + 分类讨论** 是一种非常巧妙的算法设计，它通过分类讨论解决了“恰好一半”与“严格多数”之间的鸿沟，并保持 O(1) 空间复杂度。其核心在于：  
- 理解摩尔投票的抵消原理；
- 根据目标元素可能的位置关系，将问题拆分成两个互斥的简单场景；
- 用一个循环同时处理两个场景，避免额外开销。

掌握这种思路，不仅有助于解决本题，也为处理类似“半数”或“众数”问题提供了新的视角。
