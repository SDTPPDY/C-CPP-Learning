原地哈希（也称“索引映射”或“数组归位”）是一种利用数组本身作为哈希表来存储信息的技巧，通过交换元素使得元素值与索引建立映射关系，从而在不使用额外空间的情况下解决问题。它常用于处理数组元素范围与索引范围一致或密切相关的问题。

---

## 1. 核心思想

将数组元素“归位”，即对于每个索引 `i`，如果元素 `nums[i]` 属于有效映射范围（如 `0 ~ n-1`），则通过交换将其放到 `nums[nums[i]]` 处，最终使 `nums[i] == i` 或满足特定条件。归位后，数组结构就包含了缺失、重复等信息。

---

## 2. 适用条件

- **长度与数值范围关系密切**：例如数组长度为 `n`，元素值在 `[0, n-1]` 或 `[0, n]` 范围内。
- **元素通常不重复**（或只有个别重复），否则归位时会遇到冲突。
- **需要原地操作**，要求空间复杂度 O(1)，时间复杂度 O(n)。

---

## 3. 常见操作模式

### 3.1 交换逻辑

```cpp
// 正确交换：将 nums[i] 放到它应该在的位置
swap(nums[i], nums[nums[i]]);
```

**注意**：不能写成 `swap(nums[i], nums[nums[i]])` 以外的复杂形式，尤其避免先修改 `nums[i]` 再用它作索引。

### 3.2 控制流程

有两种常见写法，本质都是**每个位置可能被多次处理直到归位**：

#### 写法一：外层 for + 内层 while

```cpp
for (int i = 0; i < n; ++i) {
    while (nums[i] != i && nums[i] < n) {   // 只在有效范围内交换
        swap(nums[i], nums[nums[i]]);
    }
}
```

- 适用于需要完全归位后整体遍历的场景（如找缺失数）。

#### 写法二：外层 while + 条件前进

```cpp
int i = 0;
while (i < n) {
    if (nums[i] == i) {
        i++;
        continue;
    }
    if (nums[nums[i]] == nums[i]) {   // 发现重复
        return nums[i];
    }
    swap(nums[i], nums[nums[i]]);
}
```

- 适用于找到目标即可提前退出的场景（如找重复数）。

两种写法等价，写法二将内层循环“展开”到了外层迭代中。

## 4. 典型应用

| 题目                                                  | 条件                                               | 核心步骤                                                                     |
| ----------------------------------------------------- | -------------------------------------------------- | ---------------------------------------------------------------------------- |
| **寻找缺失数字** (LeetCode 268)                 | 数组包含 `[0, n]` 中 n 个数，缺一个              | 归位后遍历，第一个 `nums[i] != i` 即缺失；若全归位则缺失 n                 |
| **寻找重复数字** (LeetCode 287 / 剑指 Offer 03) | 长度为 n，元素在 `[0, n-1]` 内，有且仅有一个重复 | 归位过程中若发现目标位置已存在相同值，则该值为重复                           |
| **找到第一个缺失的正数** (LeetCode 41)          | 未排序整数数组，找出最小缺失正数                   | 将值在 `[1, n]` 的元素归位到 `nums[value-1]`，再遍历找第一个不匹配的位置 |

---

## 5. 时间与空间复杂度

- **时间复杂度**：O(n)。每个元素最多被交换一次（因为每次交换都至少将一个元素放到最终位置），因此总交换次数 ≤ n。
- **空间复杂度**：O(1)。仅使用常数个额外变量。

---

## 6. 注意事项与易错点

1. **交换顺序**：必须使用 `swap(nums[i], nums[nums[i]])` 或等价写法，避免用临时变量打乱索引。
2. **越界判断**：当 `nums[i]` 超出有效索引范围（如等于 n）时，不应交换，否则访问越界。
3. **死循环**：确保交换条件能终止。例如只交换 `nums[i] != i` 且 `nums[i]` 在有效范围内，每次交换都会让至少一个元素归位，因此不会死循环。
4. **数组元素重复**：若存在重复值，归位时可能冲突，需根据题目处理（如提前返回或特殊标记）。
5. **数值范围不连续**：原地哈希要求映射关系是“值 → 索引”的一一对应，若范围不连续则需用其他技巧（如 LeetCode 41 的偏移处理）。

---

## 7. 与其他方法的对比

| 方法           | 时间复杂度 | 空间复杂度 | 适用场景                                    |
| -------------- | ---------- | ---------- | ------------------------------------------- |
| 原地哈希       | O(n)       | O(1)       | 元素范围与索引范围一致，可原地交换          |
| 哈希表         | O(n)       | O(n)       | 通用，但需要额外空间                        |
| 位运算（异或） | O(n)       | O(1)       | 仅适用于找缺失/重复等对称问题，不修改原数组 |
| 求和/数学      | O(n)       | O(1)       | 仅适用于找缺失，但可能溢出（需处理）        |

---

## 8. 总结

原地哈希是一种巧妙的空间优化技巧，通过“交换归位”将数组本身变为哈希表，适用于数值范围与索引强相关的数组问题。掌握其核心交换逻辑和循环控制，就能在 O(1) 空间内高效解决缺失、重复、最小缺失正数等经典问题。
